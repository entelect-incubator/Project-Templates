# Kestrel Port Configuration - Summary & Reference

## Your Question Answered

**Q: Does the port in Kestrel config need to match launchSettings.json?**

**A: NO.** They are completely independent:
- `launchSettings.json` → Used ONLY by `dotnet run` in development for IIS Express
- `Program.cs Kestrel config` → Used in ALL environments (dev, staging, prod, docker)

They don't conflict because launchSettings.json sets environment variables, not the Kestrel port directly.

---

## Best Practice

Instead of hardcoding port 7160, read it from configuration:

```csharp
// Current (not ideal)
builder.WebHost.ConfigureKestrel(options =>
{
    options.ListenAnyIP(7160, listenOptions => 
        listenOptions.Protocols = HttpProtocols.Http1AndHttp2AndHttp3);
});

// Better (recommended)
builder.WebHost.ConfigureKestrel((context, options) =>
{
    var port = int.Parse(context.Configuration["Kestrel:Port"] ?? "5000");
    
    options.ListenAnyIP(port, listenOptions =>
    {
        listenOptions.Protocols = HttpProtocols.Http1AndHttp2AndHttp3;
    });
});
```

## Complete Implementation

### 1. Update Program.cs
See `APPLY_KESTREL_CONFIG_TO_PROJECTS.md` for exact code

### 2. Add to appsettings.json
```json
{
  "Kestrel": {
    "Port": 5000
  }
}
```

### 3. Environment-Specific Config Files
- `appsettings.Development.json` → Port 5000
- `appsettings.Production.json` → Port 80
- `appsettings.Staging.json` → Port 8080 (optional)

### 4. How to Use
```bash
# Uses port 5000 (default)
dotnet run

# Uses port 7160 (override)
Kestrel__Port=7160 dotnet run

# Docker: Use different port
docker run -e Kestrel__Port=8080 myapp

# Production: Use appsettings.Production.json
docker run -e ASPNETCORE_ENVIRONMENT=Production myapp
```

---

## Configuration Priority (Highest to Lowest)

1. **Environment Variables** - `Kestrel__Port=8080`
2. **appsettings.{Environment}.json** - `appsettings.Production.json`
3. **appsettings.json** - Default config
4. **Code Default** - `?? "5000"`
5. **ASP.NET Core Default** - Port 5000

## Real-World Examples

### Example 1: Local Development
```bash
# Just run it
dotnet run
# Uses: appsettings.Development.json → Port 5000
```

### Example 2: Test on Different Port
```bash
# Override for quick testing
Kestrel__Port=9000 dotnet run
# Uses: Port 9000 (env var overrides config)
```

### Example 3: Docker Deployment
```bash
# Build image
docker build -t myapp .

# Run with custom port
docker run -p 8080:5000 -e Kestrel__Port=5000 myapp
# Container listens on 5000, exposed as 8080 on host

# Run production config
docker run -e ASPNETCORE_ENVIRONMENT=Production myapp
# Uses appsettings.Production.json (port 80)
```

### Example 4: CI/CD Pipeline
```bash
# In your CI/CD (GitHub Actions, Azure DevOps, etc.)
docker run \
  -e ASPNETCORE_ENVIRONMENT=Staging \
  -e Kestrel__Port=8080 \
  myapp
```
